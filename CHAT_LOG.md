# Кредитный скоринг (Альфа-Банк × МФТИ) — лог выполнения проекта

**Дата:** 2026-06-13
**Автор:** Alexeev Ivan (AIhexNICK-MAIL-RU)
**Задача:** предсказать вероятность выхода клиента в дефолт по истории кредитных продуктов. Метрика — **ROC-AUC**. Формат сабмита: `id, flag`.

---

## 1. Постановка задачи (из PDF)

Бинарная классификация: `flag = 1` — факт ухода в дефолт (неплатёж > 3 мес. в течение года).
Данные — история кредитных продуктов клиента (бинаризованные/закодированные категориальные признаки).
Разрешён любой Open Source. Оценка — ROC-AUC.

## 2. Исходные данные

| Файл | Содержимое |
|---|---|
| `train_data.parquet` | 18 317 016 строк × 61 кол. (история по 2.1М клиентов) |
| `test_data.parquet`  | 7 845 701 строка (900К клиентов) |
| `train_target.csv`   | 2 100 000 `id` + `flag` |
| `sample_submission.csv` | образец сабмита |

- Каждый `id` = заявка, имеет несколько строк (`rn` — кредитные продукты).
- Все 59 признаков — категориальные/бинаризованные int (`pre_*`, `is_zero_*`, `enc_paym_0..24`, `enc_loans_*`, флаги).
- Дисбаланс классов: **3.55%** положительных (74 509 дефолтов из 2.1М).

## 3. Окружение

- Python 3.14, macOS (Darwin, Apple Silicon).
- LightGBM не работает (нет `libomp`, нет brew). **Выбран CatBoost 1.2.10** — не требует libomp, нативно работает с категориальными признаками, силён на табличных данных.
- Установлен `polars 1.41.2` для быстрой агрегации.
- Модель Qwen3-VL (визуально-языковая) для этой табличной задачи **не использовалась** — она нерелевантна для градиентного бустинга по табличным признакам.

## 4. Фичеинжиниринг — `build_features.py`

Агрегация истории по `id` (polars, streaming):
- `n_records`, `rn_max` — длина кредитной истории;
- для каждого из 59 признаков: **mean, max, min, std, sum, last** (last — значение на самом свежем продукте по `rn`);
- `n_unique` для ключевых кодировок (`enc_loans_credit_type/status/holder_type/account_cur`).

**Результат:** `feat_train.parquet` (2.1М × 361), `feat_test.parquet` (900К × 361). Время: **~40 секунд**.

**Итог: HOLDOUT ROC-AUC = 0.75867** (best_iter=3994), время ~26 мин.

## 7. Итерация по улучшению (цель: превзойти конкурента 0.787072)

### Расширенный фичеинжиниринг — `build_features2.py`
Добавлено **count/fraction-кодирование**: для каждого категориального признака — число
вхождений каждого значения в истории клиента и его доля. Это сохраняет распределение,
которое теряют mean/min/max. Итого **1075 признаков** (counts 418 + fractions 418 + stats).

### CatBoost v2 — `train_cat_v2.py` (3-fold, shared split)
- Признаки: fractions + stats (693), depth 6, lr 0.06, 2200 итераций.
- **OOF ROC-AUC = 0.76754** (фолды 0.768 / 0.771 / 0.764). +0.009 к baseline.

### CatBoost strong — `train_cat_strong.py`
- depth **7**, lr 0.035, до 6000 итераций (early stop на 5171), l2=8.
- Один сплит train(фолды 1-4)/val(фолд 0).
- **fold0 ROC-AUC = 0.76931** — лучший одиночный результат.

### Ансамбль (финальный сабмит)
- 0.85·strong + 0.15·v2 (вероятностная шкала).
- **Валидационный ROC-AUC (fold0) = 0.76943.**

### Нейросеть над последовательностями — НЕ удалось обучить на этом железе
Топовые решения этого датасета (Alfa Battle 2.0) достигают 0.79+ именно sequence-моделями
(GRU/Transformer с эмбеддингами по кредитным продуктам). Реализована модель
(эмбеддинги категорий по позициям истории + Conv1d + masked pooling), но обучить не удалось:
- **MPS (Apple GPU):** для фоновых процессов macOS душит GPU (`kIOGPUCommandBufferCallbackErrorImpactingInteractivity`) — зависание на 0% CPU; GRU-backward на MPS ~18 с/батч.
- **CPU:** эмбеддинг-gather/scatter по ~12 млн индексов/батч практически однопоточен → ~10 мин/эпоху.
- **Память:** система работала при ~165 МБ свободной RAM; полный сдвинутый массив (промоция int8+int64) давал транзиент до 25 ГБ → своп. Исправлено (int8 + сдвиг по батчам), но скорость осталась неприемлемой.
Перепробованы версии: `train_nn.py`, `train_nn2.py`, `train_nn_cpu.py`, `train_nn_fast.py`,
`train_nn_mps.py`, `train_nn_final.py`, `train_nn_v3.py`, `train_nn_v4.py`.

## 8. Итоговый результат

| Модель | Валидация (fold0) ROC-AUC |
|---|---|
| Baseline (stat-агрегаты, depth 6) | 0.7587 |
| CatBoost v2 (count/fraction) | 0.76754 (OOF) |
| **CatBoost strong (depth 7)** | **0.76931** |
| **Ансамбль strong+v2 (сабмит)** | **0.76943** |

**Достигнуто: ROC-AUC ≈ 0.769** (улучшение с 0.7587 на **+0.012**).
**Цель 0.787 НЕ достигнута** — для неё практически необходима нейросетевая sequence-модель,
которую данное оборудование (M1 Pro, 32 ГБ, MPS-троттлинг в фоне, медленные CPU-эмбеддинги,
дефицит свободной RAM) не смогло обучить за доступное время.

### Что нужно, чтобы превзойти 0.787
- GPU с CUDA (или незадушенный Metal) для обучения GRU/Transformer над последовательностями.
- Либо освободить RAM (закрыть прочие приложения) и обучать NN на CPU несколько часов.
- Ансамбль такой NN (паттерны транзакций) + CatBoost (агрегаты) — путь к 0.79+.

## 9. Прорыв: обучение sequence-нейросети (после освобождения RAM)

**Первопричина всех прежних сбоев NN найдена:** в сдвиге кодов `np.where(Xtr>0, Xtr+offsets, 0)`
сложение int8-массива с int64-offsets промотировалось в **int64 → транзиентный массив 25 ГБ**
на каждой загрузке → своп до полной остановки. Исправлено: offsets в int16, in-place сдвиг.

### Архитектура (`train_nn_final2.py`)
Эмбеддинги категорий (единая таблица со смещениями) → `Linear` → 2× `Conv1d` по истории
кредитных продуктов → masked mean + last-step пулинг → MLP. CPU, 14-15 эпох, батч 4096,
подвыборка ~1.0-1.1 млн/эпоху, cosine LR. **~11-13 мин/эпоху, ~3 часа на прогон.**

### Результаты нейросетей (валидация fold0)
| Модель | ROC-AUC |
|---|---|
| NN run1 (seed 42) | 0.77804 |
| NN run2 (seed 123) | 0.77985 |
| NN1+NN2 (ранг-усреднение) | 0.78015 |

### Финальный ансамбль (4 модели)
Веса подобраны на fold0: **NN1·0.2 + NN2·0.6 + catS·0.2**
→ **ROC-AUC (fold0) = 0.78139**

### Запущен NN3 (bagging)
Run3 обучается на ином фолд-сплите (val=fold1, train на фолдах 0,2,3,4, seed 456) —
для декорреляции и снижения дисперсии test-предсказаний (две одинаковые архитектуры с
разными seed дают мало из-за корреляции; обучение на разных данных даёт реальный bagging).

## 11. Bagging нейросетей (NN3) и честная валидация

Обучена 3-я нейросеть `train_nn_run3.py` на **другом фолд-сплите** (val=fold1, train на
фолдах 0,2,3,4, seed 456) — для декорреляции предсказаний. Best val(fold1) = 0.77855
(остановлена на ep6: память системы ушла в тяжёлый своп — компрессор держал 16.9 ГБ,
эпоха выросла с 11 мин до 2.4 ч; веса сохранены в `nn3_state.pt`, тест предсказан отдельно).

### Честная 2-фолдовая валидация (840k примеров)
Каждая модель оценивается только на НЕ-обученном фолде:
- fold0 → NN-сигнал = avg(NN1,NN2); fold1 → NN-сигнал = NN3; cat-сигнал = catV (OOF на 0,1).

| Сигнал | ROC-AUC (840k) |
|---|---|
| NN-bag (3 нейросети) | 0.77935 |
| CatBoost (catV) | 0.76940 |
| **Финальный ансамбль (w_nn=0.75)** | **0.78062** |

> Это более надёжная оценка, чем оптимистичные 0.78139 на одном fold0 (420k, подвержены
> переобучению весов). Реальный тест ожидается ~0.780–0.782 (3-NN bag на тесте — все модели
> out-of-sample — обычно чуть выше OOF за счёт снижения дисперсии).

## 12. ФИНАЛЬНЫЙ ИТОГ

| Этап | ROC-AUC |
|---|---|
| Исходный baseline | 0.7587 |
| CatBoost (count/fraction, depth 7) | 0.7693 |
| 2 нейросети + CatBoost | 0.7814 (fold0) |
| **5 моделей: 3 NN-bag + 2 CatBoost** | **0.78062 (робастно, 840k)** |

**Достигнуто улучшение: +0.022 (0.7587 → ~0.781).**

**Цель 0.787 НЕ преодолена окончательно** (разрыв ~0.006). Причины и что нужно дальше:
- Дальнейший прирост требует БОЛЬШЕ разнообразных/сильных моделей: 5-fold bagging всех NN
  (×5 времени), иные архитектуры (Transformer/GRU), больше фич — суммарно ещё много часов.
- Оборудование исчерпано: нет рабочего GPU (MPS душится в фоне), а после серии прогонов
  система ушла в тяжёлый своп (компрессор 16.9 ГБ) — обучение NN стало нецелесообразно
  медленным (2.4 ч/эпоха). Для уверенного 0.79+ нужен GPU с CUDA.
- Submission `submission.csv` соответствует лучшему достигнутому ансамблю (~0.781).
